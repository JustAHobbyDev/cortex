# Cortex Maintainer Mode Tracking Policy

Version: v0
Status: Experimental
Inputs:
- `.cortex/` lifecycle artifacts generated by `cortex_project_coach_v0.py`
- `README.md` repository purpose and artifact admission constraints

## Purpose
Define what `.cortex/` artifacts are tracked in git when running this repository in maintainer mode.

## Scope
- In scope:
  - Tracking of `.cortex/manifest_v0.json`
  - Tracking of `.cortex/artifacts/*`
  - Tracking of `.cortex/prompts/*`
  - Tracking of `.cortex/reports/*`
- Out of scope:
  - Runtime/editor temp files under `.cortex/`
  - Any non-deterministic scratch files without governance value

## Policy
1. Track all governance-bearing `.cortex/` artifacts in git.
2. Ignore transient files (`*.tmp`, `*.bak`, swap files, OS metadata, temp dirs).
3. Treat `.cortex/reports/lifecycle_audit_v0.json` as a required maintainer signal.
4. Commit `.cortex` changes alongside spec/playbook/governance changes when they represent lifecycle state transitions.

## Determinism Rule
`.cortex/` files should be UTF-8 and deterministically serialized where JSON is used.

## Failure Modes
- Maintainer cycles run but outputs are not tracked, causing governance drift.
- Transient files pollute git history and obscure lifecycle signal.

## Immediate Next Step
Apply corresponding `.gitignore` entries for transient `.cortex` files and stage current maintainer artifacts.
